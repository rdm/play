Things I think I know

I think that this automated system is going to be creating and updating a branch named gh-pages. And, I think github has a "Jekyll" tool designed to give me an emulation of how changes to that branch will look when published as http://rdm.github.io/play (and something similar for other projects - the project name in github will replace "play" here, since this project is named "play"). Also, the author of Jekyll is Tom Preston-Werner - that might be handy for searching for docs or other information. I think Jekyll was designed to support "blog-like" publishing, and to show how different templates will look with current content.

This markdown system uses twitter-like handles (@rdm for example) to refer to github user profiles.
